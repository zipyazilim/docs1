Flura - Ultimate SaaS Agency Website Builder, Project Management System

Flura, a SaaS website builder and CRM Panel.

Installation

Set up the project using bun.

bunx create next-app@latest .

Just press "Enter" on the options.

Navigate to your project path and transfer all the files inside the "Flura" folder you downloaded to the main directory of your project. Answer the "Replace item" question with "Replace all".

Install the following modules and versions:

NOTE: Please make sure to use the same versions as in the project. Some vendors (e.g. Clerk) changed some tags in the last version. You don't want to have to change them.


"@clerk/nextjs": "4.29.1",
"@clerk/themes": "1.7.9",
"@prisma/client": "^5.13.0",
"@stripe/react-stripe-js": "2.4.0",
"@stripe/stripe-js": "2.3.0",
"@tremor/react": "3.16.1",
"@types/react-beautiful-dnd": "^13.1.8",
"prisma": "^5.13.0",
"uploadthing": "^6.10.0",
"All shadcn components",

You will find a copy of the modules to install in the package.json file.

Stripe Settings

Navigate to the .env file to adjust Stripe settings.


Webhooks:
{yourwebsiteURL}/agency
{yourwebsiteURL}/subaccount

Paste the Stripe API Key, API Secret and Webhook Secret values to the .env file.

Stripe Package Settings

  • Go to Stripe Dashboard.
  • Add a Product.
  • "Add Price" in Product.
  • Set its name to Basic (or your package name.) Enter other details and select a renewal period.
  • Copy the API ID of the price value you created and paste it back into your application, opposite the "priceId:" value of the package named "Basic" in the src>lib>constants.ts file.
  • Then go to prisma>schema.prisma in your application and paste line 399 - 400 (under "enum Plan").
  • Do the same for the Unlimited package. Create a Stripe price, this time paste it in the field with the value "Unlimited SaaS".
  • Now create a new product in Stripe. (Note: Not a new price tag, but a separate product.)
  • Name it "Support Package" (or whatever name you want).
  • Create a new price. Set the price and name of your service.
  • Copy the value starting with "prod_" at the top right of the page. (NOTE: make sure not to copy the value price_. Product is at the top right of the page).
  • Now go to src>lib>constant.ts and paste it opposite the id: value in the field titled "Priority Support" on line 61. Make sure not to delete the quotation mark.

Clerk Settings

We will use the Clerk tool to log in to the application.

  • Log in to your Clerk account.
  • Create an app with Production mode.
  • Go User+Authentication>Email, Phone, Username
  • Open "E-Mail address" and set required.
  • Close "username" tag.
  • Open "Password" and "Email verification link".
  • Open "Name".
  • Go to "API Keys" tab and copy your Clerk API Keys.
  • Go back Visual Studio Code and go .env file. (PS: We are added a env.example file. Please copy and paste your .env file.)
  • Paste Clerk API Keys in the corresponding field.

Uploadthing Settings

We will use Uploadthing for media storage.

  • Go to your Uploadthing account.
  • Add and App.
  • Copy your APP ID and Secret key.
  • Paste your keys into the .env file.

SQL Export

After completing your project, you should output it to the database. We recommend Railway or Planetscale for this. Railway was used in this project. If you don't know how to install Railway on your computer, have a simple look at Railway Docs. Install it with Terminal and send a db push after making database settings from the .env file. Now you are ready!
